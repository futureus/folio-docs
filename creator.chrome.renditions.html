Chrome renditions

All chromes are created to a specific size, so you can't have a single chrome that will support all devices and orientations. You can however create multiple chrome files for each rendition type that load per device.

To create a chrome for a specific device and orientation, create a new file in your issue's directory named relevant to the orientation you desire. Here is a full list of the available chrome files you can use:

  • chrome.xml - If you're only using one rendition, then just use this file. You don't need to specify a specific chrome file if you don't have multiple renditions. On a side note; if you don't create the relevant files for a rendition, the app will default to this file.
  • chromePadPortrait.xml - Specifies the iPad portrait orientation chrome. 768 X 1024 (1004, the status bar at the top is 20 points tall).
  • chromePadLandscape.xml - Used for landscape iPad orientations. 1024 X 768 (748).
  • chromePhonePortrait.xml - This chrome will only load for iPhone devices in the portrait orientation. 320 X 480 (no status bar on the iPhone).
  • chromePhoneLandscape.xml - The iPhone in landscape mode. 480 X 320.
