Building an issue

Once you've completed your issue you'll need to export it out of FutureFolio Creator, this process is called building. When building, FutureFolio Creator will automatically take all used images, movies and media assets, scan your body copy and grab any files from there and finally check for any files referenced in your AdditionalBuildFiles file, copy them, compress, resize, create retina and non-retina versions of them and bundle them together as a single issue archive.

A built archive can be placed in FutureFolio Viewer (see "Viewing in FutureFolio Viewer") or submitted to your account manager to add the issue to your FutureFolio app.

In this section you'll learn how to setup your issue for building and what the build process is actually doing.

  2. Issue version support - Understand how FutureFolio handles exported issue versions and corresponding app versioning.

  4. Partial builds - Enable fast builds for interim testing.
  5. Cover page - Selecting the page to use as the issue cover.
  6. Understanding the build report - Understand the output file created for each build.
