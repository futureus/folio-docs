Particle Effects

FutureFolio can create special effects commonly seen in video games, such as fire, smoke, fog, steam, rain and snow. These effects are called particle effects, because the special effects system is made up of hundreds (or perhaps even thousands) of tiny particles that move around the page in a way that, combined, looks like the effect you're trying to produce.

Particles are immensely powerful, which means there's a learning curve required to take full advantage of them. You have complete control over how the particles behave, which means it's possible to create your own custom effects very easily.

The best way to create particles in Creator is to use the Particle Effects editor window, which is located under the page editor – tap the Effects button then choose Particle Effects.

Below is a description of what the various controls do. Any value marked "range" allows you to specify a range of values (for example, a Red value of 128 and a Red Variation value of 10 means that any red value between 108 and 138 can be used).

  • X - the horizontal position where the particle system should be created.
  • Y - the vertical position where the particle system should be created.
  • Width - the width of the particle system; this is not used if you choose Line shape.
  • Height - the height of the particle system; this is not used if you choose Line shape.
  • Shape - whether particles should be created from a single point, a rectangle or a line.
  • Mode - "normal" mode will show overlapping particles with no special blending; "additive" will increase the brightness of overlapping particles that have have an alpha value below 1.
  • Birth Rate - how many particles to create every second. Try to keep this as low as possible.
  • Lifetime - how long in seconds the particle system should continue to make particles.
  • Particle Lifetime - how long in seconds each particle should live
  • Particle Lifetime Range - how long in seconds to vary the length of each particle's life
  • Images - a collection of images to use for your particles; one of these will be randomly chosen for each particle.
  • Container Layer - if specified, particles will be created inside this layer on your page; this allows you to control their exact position above or below other layers.
  • Red - the starting red colour of each particle, specified as 0-255 (0 = no red; 255 = max red)
  • Green - the starting green colour of each particle
  • Blue - the starting blue colour of each particle
  • Alpha - the starting alpha value of each particle (0 = transparent; 255 = fully opaque)
  • Red Range - the amount that the red colour should vary from particle to particle
  • Green Range - the amount that the blue colour should vary
  • Blue Range - the amount that the blue colour should vary
  • Alpha Range - the amount that the blue colour should vary (negative alpha values and alpha values over 255.0 are forced into the range 0.0 - 255.0.)
  • Alpha Speed - the amount that the alpha value of each particle should change over its lifetime. This should be specified as a value between -255 and 255. See below for details.
  • Velocity - how fast each particle should move
  • Velocity Range - the amount the speed should vary from particle to particle
  • Horizontal Acceleration - how much the horizontal speed should change over time (see below for details)
  • Vertical Acceleration - how much the vertical speed should change over time (see below for details)
  • Direction - the direction, in degrees, to create particles, where 0 is directly upwards and 90 is 90 degrees to the right.
  • Direction Range - the amount of varation of particle creation direction, in degrees; a value of 90 will create a 90-degree cone around your direction.
  • Spin - the number of rotations each particle should perform over its life time; specify negative values for anti-clockwise rotation.
  • Spin Range - the amount to vary the spin value from particle to particle
  • Scale - how large each particle should be created; the default is 1, which means its normal size.
  • Scale Range - how much to vary the size of each particle.
  • Scale Speed - how much to modify each particle's scale over its lifetime.

When you've created your particle system, enter a filename into the box in the bottom-left corner of the screen, and tap Save. You can also load a pre-saved particle system for editing by tapping the Load a Saved Effect button.

Alpha Speed

This value affects how your particles change their alpha value during their lifetime. If, for example, your particle is created with an alpha of 128 (50% transparent), setting an alpha speed of -128 will make the article fade out to be invisible as it ages.

Before you use your particle system, we recommend you tap Preview to ensure that it looks and works as you intended.

Once your particle system has been created, you can create it using the "Create Particles" behaviour and specifying your particle filename as the behaviour's parameter.

If you wish to have one particle image be used more than the others, simply specify it more times. For example, adding Image1 three times and Image2 once will cause Image1 to be used 75% of the time.

Particle performance

Particle effects are rendered on the device's graphics card, which means they are very fast. However, there are three exceptions:

  1. Having a high birth rate and/or having large particle graphics.
  2. Using multiple particle systems on a page, or on adjacent pages.
  3. Running in the iOS Simulator.

We strongly recommend you avoid using multiple particle systems on adjacent pages - leave at least a one-page gap between particle systems to ensure they all run at maximum speed.

Although the iOS Simulator does a very good job of simulator a real iPad, particle systems run slower in the virtual iPad than they do in a real iPad.
