Importing Issues

You can import any given issue in your /Simulator/EditingIssues directory into another issue.

This feature can be useful, for example, if you want to work on different parts of your issue in isolation and/or by different users - and bring it all together at a later time.

To do this, open the issue which is to be the result of the merge and then click on the Import button, selecting the issue to import.

Importing an issue

The selected issue will be read, and its pages and sections imported into the currently viewed issue.

For each section in the imported issue, if the viewed issue contains a section of the same name, the pages from the imported section will be added to the end of the corresponding viewed issue's section.

If a given section name exists in the imported issue but not in the viewed issue, the imported section will be added to the end of the viewed issue's sections.

Page names

When importing one issue into another, all page names need to be unique, across both issues.

If any given page name is used in both of the issues, the import request will fail and display an error message.

Import failed - common page names

The import process will copy all of the imported issue's assets into the viewed issue's directory. It will also copy the preview images for the imported pages.

Duplicate assets

For each asset in the imported issue's directory, if the viewed issue's directory contains an asset of the same name, the copy process will skip that asset - the asset in the viewed issue will not be overwritten.
