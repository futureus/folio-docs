Set a Variable

This lets you set a named variable, either to a specific value (eg 5000 or "Hello"), to a relative value (eg +5000 or -1), or to a list of values (eg "Gears of War", "Halo" and "Mass Effect").

For example: score=5000 will set a variable called "score" to be 5000. If you then say score+5000, it will set score to be 10000 - FutureFolio will take its current value of 5000 and add another 5000 to it.

You can also specify negative values, eg score-5000, which will subtract 5000 from score.

If you wish to make lists of items, such as a wish list or shopping basket, then you can use + and - with text surrounded by quotes. For example, wishlist+"Gears of War" and wishlist+"Halo" will result in both those items being in the wishlist variable. If you use - followed by some text, eg shoppinglist="carrots", all instances of that text will be removed from the list.

When a variable is changed, any Story layers that use variables will be reloaded.

You can assign text to variables simply by typing it in. For example, if you ask the user to choose which games console they want to read about, you could store console=PS3 and check that value later.

If you intend to use your variables to print lists of items, you should know that your variable names can and indeed should be named in a way that means something to the user. That is, "Shopping List" is preferable as a variable name to "shoppinglist".

Parameter: the name of the variable to set, an equals sign, followed by either an absolute value (name=hello); alternatively, the name of the variable to set followed by a relative value (score+5000; score-5000).

Variables are shared

Variables are accessible across all issues of your magazine, which is helpful because it means you can read data across issues, but does mean that issues can tread on each others' variables unless you are careful.

If you want to avoid accidentally treading on variables, put your issue number or some other unique code at the start of your variable names, eg 194Score.
