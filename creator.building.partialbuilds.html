Partial builds

By default, an issue build performs the following actions:

  1. Clears the results of any previous build.
  2. Resize and compress all the image assets you have used.
  3. Copy all assets (images, tables, animations, particle effects, etc) to your build directory.
  4. Copy in your page data (layout, stories and behaviours).
  5. Compress the entire issue and produce a single zip file.

When Partial Build is enabled, steps 1-3 are skipped, which will result in your issue build taking substantially less time than a full build. However, there are a number of provisos:

  1. If you have changed an asset's data (eg changing the contents of a movie, image or animation), it will be ignored.
  2. If you have changed an image's size, it will be ignored.
  3. If you have changed your JPEG quality, it will be ignored.
  4. You must have performed at least one full build before you can perform partial builds.
  5. Your build report will not report any image problems.

Final builds

Never use a partial build for your final issue build, because your zip may include stale content and you may miss important build report warnings.
