Slideshow

The slideshow image layer type is a series of pictures which the user can manually swipe through.

Specify the images to display in the slideshow with the Filenames button (see "Importing Filenames").

If you wish, you can add additional image layers to show left and right arrows to indicate to the user that they can swipe to see more. If you have chosen to do this, you can enable a feature on the slideshow layer where Folio hides the left arrow when the first image is showing, and hides the right arrow when the last image is showing. To enable this feature, enter the left and right arrows' layer names into the Arrow layers dialog.

By default, the slideshow will scroll in a "paging" manner. As the user swipes and lets go, the slideshow will dock to the nearest unit, where the unit is defined to be the width of the slideshow layer. To change the scroll type to a smooth scroll, where the slideshow does not dock as the user lets go, change the Smooth scroll control to "OFF". You can also set this to hidden, this will use paged scrolling without the paging dots at the bottom of the slideshow.

The default paging unit for a slideshow is the width of the slideshow layer. You can change this using the Scroll % control. Entering a value of less than 100 will result in a smaller paging unit, so that the next image is partially visible at the edge. This can be useful to show the user that there is more content which they can scroll to see. If your image matches the aspect ratio of the slideshow layer, this will result in some image overlapping. However, if the image has a different aspect ratio and you choose the "Fit" option (see "Fit vs Stretch"), your images will be displayed to the size of the slideshow, but with the aspect ratio maintained, so you will have space either side of your image. This can be used in conjunction with the Scroll % control to give the results shown in "Scroll percent in slideshows".

The figure below shows a slideshow which is the same width as the page. The images which make up the slideshow are half of that width. With a scroll percentage of 100, the next image is not visible. At 70, the next image can just be seen, 60 shows a large chunk of it, and 50 lines it up to the edge of the current issue, as in this example, the image width is exactly half the slideshow width.

When adding a file to a slideshow, the Add remote... button can be used to add a movie into the slideshow. To use this feature, enter the filename (for local movies) or URL (for streamed movies) of the required movie in the dialog that pops up.

Naming conventions

  • Simple You can associate layers with each of the component images in a slideshow by naming them the same as the slideshow layer name, followed by a dash and a number. As the user swipes through the slideshow images, these specially named layers will be shown and hidden at the appropriate times. For example, name your slideshow layer main, then create layers named main-1, main-2 and so on, hiding each one except the one you start from (main-1). As you swipe through, main-1 will be shown when the first image displays, main-2 when the second image displays, and so on. If you are using the behaviour "Download all streamed movies" in your issue, you may wish to supply a different URL to download any movies in your slideshow from. For example, you may wish to stream an SD version, but allow the user to download an HD version. To specify a separate download URL, enter it after your streamed URL, separated by a comma or pipe character |, when adding the movie to the slideshow. E.g. http://streammoviefrom.mp4, http://downloadmoviefrom.mp4 or http://streammoviefrom.mp4|http://downloadmoviefrom.mp4 Note that commas entered here will be converted to pipe characters as commas are already used to differentiate the slideshow items from one another.
  • Advanced You can extend the simple naming convention to show multiple layers for each slideshow image. Renaming our layers to main-1:5, main-2:4 and main-6:10 will now behave as follows. The layer main-1:5 will be visible whilst the user swipes through slides 1 to 5, layer main-1:5 and main-2:4 will be visible for slideshow images 2 to 4, and main-6:10 for layers 6 to 10.
