Conditions

All image and rendition behaviours can have conditions attached to them that restrict when they can be run. By default, no conditions are applied, which means that your behaviour will always be run. To attach conditions to a behaviour, tap the settings icon next to the behaviour.

If you have attached one or more conditions to a behaviour, all of them must be true in order for the behaviour to run.

To get started, click "Add…". Each condition is made up of three parts: a variable name, a check type, and a value.

  • equals - This is true if the variable is exactly equal to the value. This can be used for numbers (eg score = 4000) or text (eg console = ps3).
  • does not equal - This is true if the variable is not equal to the value. This can be used for numbers or text.
  • is greater than - This is true if the variable is greater than (but not equal to) the value. This can be used for numbers only.
  • is greater than or equal to - This is true if the variable is greater than or equal to the value. This can be used for numbers only.
  • is less than - This is true if the variable is less than (but not equal to) the value. This can be used for numbers only.
  • is less than or equal to - This is true if the variable is less than or equal to the value. This can be used for numbers only.
  • overlaps layer - This is true if the layer name specified under "variable" overlaps the layer name specified under "value" on your page. This can be used with layer names only, and only affects visible layers.

Advanced layer overlaps

When working with the "overlaps layer" condition, you can specify both the variable and value as comma-separated lists of layer names, and if any of those layers overlap then the condition is true. For example, specifying "alien1, alien2, alien3" as the variable and "human1, human2, human3" as the value will evaluate to be true if any alien overlaps any human.

Additionally, if multiple layers have the same name, all of them will be checked for collision.
