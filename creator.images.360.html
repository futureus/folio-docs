360

The 360 image layer type is a sequence of images which the user cycles through by swiping. A single swipe results in multiple image changes, which continue, decelerating, after the user lets go, finally coming to rest a short while later.

360s are designed for image sequences such as a series of pictures of an object from different angles, all the way around. This creates the illusion of a 3D model of the object which the user pans around by swiping.

Effectively, a 360 image layer gives the illusion of a user-controlled animation. While most effective as a 360-degree animation of an object, it can also be used to good effect in many other types of animation, where the images are seen as keyframes.

Specify the sequence of images to display in the 360 with the Filenames button (see "Importing Filenames").

You can control the response speed of your 360 layer by setting the Swipe speed value. The default is 1. To make the image changes more responsive, increase this number. To make them less responsive, decrease it.

Just like with slideshows, you can associate layers with each of the component images in a 360 by naming them the same as the 360 layer name, followed by a dash and a number. As the user swipes through the image sequence, these specially named layers will be shown and hidden at the appropriate times.

For example, name your 360 layer main, then create layers named main-1, main-2 and so on, hiding each one except the one you start from (main-1). As you swipe through, main-1 will be shown when the first image displays, main-2 when the second image displays, and so on.
