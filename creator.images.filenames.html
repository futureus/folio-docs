Importing Filenames

To specify the image(s) to display in the image layer, click on the Filenames button. This will bring up a menu listing any images already assigned to this layer.

If there are no files selected, a list of image files which are currently in your issue's directory in /Simulator/EditingIssues/ will appear. Click on the image which you want to add, using the search bar to help locate it if necessary. If you already have an image or multiple images selected, to add another file, click on the Add file... button. This will bring up that same list.

To return to the list of assigned image files without adding a new one, click on the Edit slideshow button.

The ordering of images assigned to this image layer can be rearranged by dragging the Move icon on the right of the list.

Images can be removed from the image layer by clicking the red Delete button next to the assigned image in the list.

Add remote

The Add remote... button is used to add a remote movie to a slideshow. This feature only works with the Slideshow type and will be covered in "Slideshow". Note that this functionality is for remote movies, not remote images.

For some image layer types, you may wish to import a sequence of images. In these cases, you should name your images by the same filename, but with a dash and sequential numbers at the end, e.g. file-1.jpg, file-2.jpg etc. Also supported is file-01.jpg, file-02.jpg. Note that all supported image formats may be sequenced in this way.

To import the full image sequence, select the first (ending in 1). If a sequence is detected, you will be asked if you wish to import the full sequence or just the selected image.

Supported file formats for image layers are:

  • jpg
  • jpeg
  • png
  • mp4 (slideshow only)
  • mov (slideshow only)
  • m4v (slideshow only)

Retina

In order for your images to look sharp on retina displays, two images are needed - the normal one and then a second one which is double the dimensions, named with the same name, but suffixed "@2x", e.g. myImage.jpg and myImage@2x.jpg. Retina displays will choose the larger image, while non-retina displays will show the smaller.

If the source image file you provide is large enough, Folio will create the double-sized (retina) version for you at build time - it will scale the source image down to double the required display size (or as large as it can without upscaling) and save this to the build directory as myImage@2x.jpg, then scale to the required display size and save this as myImage.jpg. Therefore, wherever possible, you should provide your images at a size at least double their required display size.

If you have @2x images in your issue directory for any reason, you should never specify them as the file to use in the image layer. Always specify the standard-sized image, and the correct image will then be chosen at runtime.

Note that for standard image layers, you do not need to provide the @2x image yourself - simply supply your source image at double the display size and Folio will do the work for you.
