Scratch

Scratch image layers behave like a scratchcard - you specify a top image and an optional bottom image. The top image is then "scratched" away by the user to reveal the bottom image.

Scratch image layer

Use the Filenames button to specify the top, scratchable image first, then optionally the bottom image second (see "Importing Filenames"). If no bottom image is supplied, the top image will be scratched away to reveal whatever lies behind the layer.

Once an area has been scratched away, touches by the user are passed down to the next layer, so if the user swipes on a scratched-away area, the page-scroll action will occur, unless you have placed an interactive element underneath.

You can specify the extent of the area which is scratched away by the user with the Size control. The figure below shows sizes of 50, 100 and 200.

You can also choose between a hard or a feathered edge on the scratched area with the Feather edges toggle. The figure below shows the hard edge in the top images, and the feathered edge in the bottom images.
