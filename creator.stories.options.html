Story layer options

Tapping the Behaviours button allows you to set a number of options affecting the behaviour of a story layer, as seen in "Story layer options". The settings are:

  • Allow user to scroll: determines if the story box should be scrollable. This is best switched off for static content that fits in its entirety in the box. Text that has been laid out to fit perfectly into a layout may benefit from a non-scrolling box.
  • Text is user scaleable: determines if the user should be allowed to change the font size. User-scaleable text requires design consideration from the beginning, and normally benefits from being placed in a scrolling box.
  • Enable edge bounce: edge bounce is a feature used in many places in iOS, letting you seemingly "scroll beyond the bounds" of a box, and when you let go, the content "snaps" back into bounds. With edge bounce switched off, scrolling will stop dead at the box boundary.
  • Scrollbars: change the colour of the scroll bar of the story box.
  • Floats over page: normally, a story box is "glued" to the page, meaning that if the page it is housed on moves, the story box moves with it. The Floats over page allows you to alter this behaviour, going from normal scrolling to "pinned static", the latter giving the appearance of a "floating" box under which the page moves on scrolling. Judicious use of this feature can create a "faux" parallax effect on scroll.
  • Dynamic movement: if enabled, this story will respond to accelerometer or physics events. Please consult the chapter Dynamic movement for the specifics relating to Dynamic movement.
  • Gradient values: if set, a gradient overlay will be placed over the story. The direction can be set either vertically or horizontally and the start and end positions of the gradient fade are specified as percentages relative to the size of the story container. For example, a vertical gradient with a start value of 20% and an end value of 50% would look like Story gradients.
