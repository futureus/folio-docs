Optimising your PDFs

To ensure your digital replica will run on all the currently supported iPhone and iPad devices and display correctly, your pages PDF files will need to be exported correctly.

Digital replicas are created by exporting each page of the publication as a separate PDF file. For optimal performance you will need to export your PDFs with the following setting:

  • Export a single page PDF of every page of the publication
  • The pages should be laid out as single pages and not spreads
  • Starting from 0, pages need to conform to the page-0.pdf naming convention (page-0.pdf, page-1.pdf, page-2.pdf....etc)
  • Export to PDF version 1.4
  • The PDF should not be flattened or repurposed from a flattened file
  • Colour and greyscale images: 150ppi, maximum quality JPEG compression
  • Monochrome images: 300ppi, CCITT Group 4 compression
  • All colours should be converted to sRGB
  • No spot colours
  • No printers marks
  • All print-specific design elements should be omitted
  • Add hyperlink boxes if you wish urls etc to be clickable
  • (Optional) Encrypt each PDF with an unified password
    • Requires a password to open the document
    • Encrypt all document contents
    • If using Acrobat Professional choose Compatibility: Acrobat 7 and later, encryption level 128-bit AES
    • If using Acrobat Professional this can be done by creating a security policy – this is useful because the person applying the security policy doesn’t need to know the password

Colour profiles and you

CMYK is a print colour profile, it will not display correctly on an iPad or iPhone, you will get a dodge effect. To avoid this, ensure you are using the RGB colour profile.

iPhone and iPad devices have a limited amount of hardware resources, this is even more true in lower end older devices. These settings prevent slow load times, potential memory warnings and reduce your issue download sizes while maintaining amazing quality.
