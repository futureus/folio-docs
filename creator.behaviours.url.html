Load a URL internally / externally

Use these behaviours to link the user to a web page by providing the URL as a parameter. An internal URL loads the web page over the existing issue, and the user returns to your app by tapping Close. An external URL will quit your app and fire up the device's web browser. Before external links are opened, the user will automatically be prompted whether they want to open the link.

If you wish to direct the user to an app on the App Store, use an external URL - FutureFolio will detect the App Store URL, prompt the user correctly, then launch the App Store directly.

Internal links allow you to customise the bar at the top of the internal browser window. This is done by entering certain keywords before the URL in the parameter field. The keywords (listed below) must appear before the URL and must be comma separated - e.g. safari, title, http://www.google.com

  • safari - Show the Open in Safari button
  • back - Show the Back button
  • forward - Show the Forward button
  • refresh - Show the Refresh button
  • title - Show the browser title
  • space - Separates left items from right items (see below)

Any toolbar items specified before "title" or "space" will be displayed (in the specified order) on the left side of the toolbar. Any items specified after either of these two keywords will be displayed on the right of the toolbar. See examples below.

Parameter of: back, forward, title, refresh, http://www.google.com

Parameter of: safari, space, refresh, http://www.google.com

URL format

URLs must include the http:// or https:// portion

Parameter: the URL of the website to load, including keywords where relevant.
