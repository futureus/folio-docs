Creating page thumbnails

PDF files can take a second or two to render completely, however images can display nearly instantly. But creating low resolution thumbnail files FutureFolio Viewer disguises this delay by displaying something that closely resembles the final page. If you do not have these thumbnail files, your issue will not work.

Like with your PDF files, there are a number of settings that need to be followed in order to create your digital replica:

Thumbnails need to match up

Remember that your thumbnails are the interim display method before the PDF pages have fully loaded. Your thumbnails need to both match the corresponding PDF of the same name and not contain any crop marks or bleed edges that would result in the thumbnail being smaller or bigger than the PDF page.

  • Create an image file for each page of your publication.
  • Starting from 0, thumbnails need to conform to the page-0.jpg naming convention (page-0.jpg, page-1.jpg, page-2.jpg....etc) to match the PDF files.
  • Image files must be saved in the JPEG format using the maximum quality compression (using the .jpg filename extension).
  • The maximum height of your thumbnails needs to be 256 pixels tall. The width can be variable.
  • You must use the colour profile RGB
