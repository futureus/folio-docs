Sequences

Animation sequences allow you to create animation chains that execute one after another, and are the best way to create complex animations where one movement should execute only if another has finished.

Animation sequences are created in two parts: naming animations and using the onFinish attribute for animations.

A named animation looks like this:

<animationBlock>
	<animation delay="2" duration="3" sequenceName="myanim">
		<layer name="Picture1" alpha="0" />
	</animation>	
	
	â€¦more animationsâ€¦
</animationBlock>

That animation will not be executed until something else in the same animation script tells it to run.

Sequenced animations won't auto-execute

Do not give your animations a sequence name if you want them to run automatically. Any animation with a sequence name will not be executed until it is forced to execute by another animation in the same script.

Execute a named animation is done by adding an onFinish attribute to an animation, like this:

<animationBlock>
	<animation delay="1" duration="1" onFinish="myanim">
		<layer name="Picture1" scale="2" />
	</animation>		
	
	<animation delay="2" duration="3" sequenceName="myanim">
		<layer name="Picture1" alpha="0" />
	</animation>	
</animationBlock>

You can trigger multiple animations using onFinish, either by specifying multiple sequence names separated by commas (eg, onFinish="a, b, c") or by giving more than one animation the same sequence name. For example, if you use onFinish="myanim" and there are two animations named "myanim", both will be executed.

Delays in animation sequences are based on the time they were started, rather than the time the whole animation script started. As a result, the above animation script will execute like this:

  1. After one second, make Picture1 scale up to 2x its size over a period of one second.
  2. When that has finished, wait a further two seconds, make Picture1 fade out over a period of three seconds.

Although you could do that without sequences, using them guarantees you precise timing.
