Creating your chrome

Upon creating a blank issue, FutureFolio will automatically generate a chrome.xml file. By default this chrome will load the standard built in chrome that presents a simple constant top bar with a Close button.

To customise your chrome you will need to open the chrome.xml file in your preferred text editor and change the code.

All chrome files star and end with chrome tags, like so:

<chrome>	
	// Your chrome tags here
</chrome>

Inside the chrome tag there are two things you can set:

  1. default - if set to true, ie default="true", and you have created no chrome bars, this will place a dummy chrome on your page to help you test your magazine without first having to design a chrome.
  2. dimAmount - set this to a value between 0 and 1 to control the amount the page should be dimmed when the chrome is opened, where 0 is no dimming at all (the page remains active) and 1 will black out your page entirely.

Inside your chrome tags you need to create bar tags, these will equate to a physical bar that will display in your document.

bar tags have several attributes, some of these are required so that your issue knows where to display your bar and how. Here's a full list:

  • name (required) - This is the unique name of your bar, you can use spaces, but no special characters.
  • xoff (required) - This is the default x (horizontal) position of your bar when it is off (default, unless initialState is set).
  • yoff (required) - Like xoff, this is the initial y (vertical) position of your bar when the bar is off.
  • xon (required) - When your bar is on, this is the x position where your bar will animate to, if you have initialState set to on, then the bar will automatically be set to this.
  • yon (required) - Just like xon, this represents the y position that the chrome needs to animate to when it is activated.
  • width (required) - You need to set the size of your bar, if it's full screen you will need to set this value to the full width of the device. Some titles will use multiple bars that slide in, this is why you need to set the size of each bar.
  • height (required) - Same with the width attribute, this either needs to be the exact height you want the bar to be or the full device size.
  • swipeEnabled - This feature adds up and down swiping to toggle the bar. The user has to be swiping within the on or off area of your bar for the effect to occur, if the entire bar is off the screen this will not work. To enable this, set it to y. To disable this, just do not add the attribute (if this item is set, the app will class it as a y).
  • background (required) - Each bar needs a background image, if you want to handle the imagery with your internal buttons and images, then you may wish to set this as a transparent png file.
  • alphaOff - For a fading bar, you will need to set this value, it's the off state visibility. This requires a number between zero and one, you can use decimal places for mid points, zero being invisible and one being fully visible (so 0.5 would be 50% visible). ideally you'll want to set this as 0 (invisible).
  • alphaOn - If you've set alphaOff, you'll need an on state, this is what the bar will animate to when the bar is toggled on. It uses the same values as alphaOff
  • shouldScroll - If you want your bar to be scrollable, in other words you want to have items in your bar that go further down than the height of your bar, then set this option to y. To disable this, just do not add the attribute (if this item is set, the app will class it as a y).
  • scrollColour - You can change the scroll indicator colour if you want it to be a different colour than the default black one. This ideal for dark coloured chromes. The options are: white or none (For a black indicator, just omit this attribute).
  • initialState - If you want your bar item on by default, you need to set this attribute to on. You can either omit this or set it to off if you do not want to have the bar on by default.
  • appearSound - When activated the bar will play the sound provided. Please note that you can only play audio files in the wav, caf, mp3, aif and aiff file format.
  • disappearSound - If the user closes the bar, this sound will play.

Remember to include your image files in your issue's directory

All images referenced in your chrome need to exist in your issue directory, just like any other image you may use in your issue.

Here's an example of a bar in action.

<chrome>	
	<bar name="top chrome bar" xoff="0" yoff="-150" xon="0" yon="0" width="768" height="200" swipeEnabled="y" background="top_chrome_bar.png">
		// items in your bar here
	</bar>
</chrome>

As you can see, this is a top bar that start 150 points above the top of the screen, showing the bottom 50 points on the screen (here we could display a toggle button). When the bar is on, it moves to the zero y position, showing the full bar.

Inside a bar you have three item types you can add, they are:

  • togglebutton - These are special buttons that toggle the entire chrome. This is essential if you haven't set the initialState attribute of your bar to on.
  • button - The bread and butter of every chrome are the buttons, they activate the various functions in your chrome, from jumping to pages, sections or even sharing text on social networking services.
  • scrubber - A scrubber is a visual navigation device, it's a scrollable area that contains thumbnails of each page. Tapping on a page thumbnail jumps to that page.

All these bar items share some common attributes, they are: x, y, width and height.

The Scrubber has two unique attributes:

  • padding - this sets the number of points of padding each page thumbnail image has on each side of it.
  • centrePages - if this attribute is set or set to "y" it will place the current page in the middle of the scrubber bar, not at the far left (default).

Here's an example of a scrubber bar item in action.

<chrome>	
	<bar name="top chrome bar" xoff="0" yoff="-150" xon="0" yon="0" width="768" height="200" swipeEnabled="y" background="top_chrome_bar.png">
		<scrubber x="20" y="40" width="728" height="80" padding="4" centrePages="y" />
			
		// other items in your bar here
	</bar>
</chrome>

The above creates a centralised scrubber bar that is 40 points down from the top of the bar and stops 20 points from the left and right.

Togglebutton and button items share two attributes that other items do not have:

  • x - this is the horizontal position relative to the top left corner of the bar the item is nested in. So to place an item 80 points in on a screen while in a bar that's 40 points in, you only need to set it's x value to 40 and the bar is already 40 points in from the left.
  • y - again like x, only this is the vertical position on the item relative to the top of the bar it is nested in.
  • up - both button and toggleButton items act like generic interface buttons. The "up" state is the buttons initial state or unselected state. Enter the name of the image file for your button as this attribute.
  • down - when selected or tapped, this is the image state that the button will change to. This needs to be a separate button image if you want it to change on selection. This attribute is optional, you do not require an image for the down state.

With togglebuttons, the up and down attributes can have multiple image filenames separated by a colon (,) character. The image will change based on the section number that you are in.

You can provide a fully transparent PNG image file if you want a tappable region without an image. This can be useful if you want to hide your chrome completely when it is turned off but still enable the user to activate the chrome.

Let's see togglebutton items in action.

<chrome>	
	<bar name="top chrome bar" xoff="0" yoff="-150" xon="0" yon="0" width="768" height="200" swipeEnabled="y" background="top_chrome_bar.png">
		<togglebutton x="0" y="150" width="768" height="50" up="open_arrow.png" down="close_arrow.png" />
		<scrubber x="20" y="40" width="728" height="80" padding="4" centrePages="y" />
			
		// other items in your bar here
	</bar>
</chrome>

In the example, the togglebutton item is always tappable, it's in the bottom 50 points of the top bar.

Regular buttons have further attributes that can be utilised in your chrome. They are the most versatile items in a chrome and can perform many tasks from jumping to a page to closing the entire issue.

Standard button items can have the following attributes:

  • pageSection - this is similar to how the togglebutton item handles section image changes. You can set this attribute to the name of a section, when the user is in this section the button will switch to it's (selected) down status.
  • tapEffectType - You can set your button to use a tap effect, a list of these effects are below.
  • tapEffectData - In most cases this is optional, it is only required if your tap effect needs a value, some data or a command.
  • sound - When tapped the button will play the sound provided. Please note that you can only play audio files in the wav, caf, mp3, aif and aiff file format.

Here is a list of available tap effects and the data required (if applicable) for each one:

  • TAP_JUMPTOSECTION - This jumps the user to the section entered in your tapEffectData attribute.
  • TAP_JUMPTOPAGE - Like jump to section, this jumps the user to a specified page name in your tapEffectData attribute.
  • TAP_PREVIOUSPAGE - Just like an internet browser's Back button, this sends the user to the last page they were on.
  • TAP_NEXTPAGE - Just like an internet browser's Forward button, this sends the user to the last page they jumped backwards from, almost like an undo for the previous page feature above.
  • TAP_TOC - Opens a searchable table of contents from this button. The user can filter down the pages or scroll through the whole issue.
  • TAP_CLOSEISSUE - Closes the issue and returns to the main container app. In FutureFolio Creator it will close the test.
  • TAP_INCREASEFONT - Any live text (not images) will increase in font size. This is global for the entire container app. Please bear this in mind.
  • TAP_DECREASEFONT - You will need this if you wish to enable increase font size, it enables the use to shrink the live text inside your issue.
  • TAP_RESETFONT - Sets the font size back to it's default.
  • TAP_SHAREOPTIONS - Opens a pop over containing any of the three sharing options available in app. You need to set the tapEffectData to any of the following, comma separated for multiple selections, you can have; facebook, twitter or email.
  • TAP_EXTERNALURL - Will open safari with the url provided in the tapEffectData attribute

Here's an example of a full bar in action:

<chrome>	
	<bar name="top chrome bar" xoff="0" yoff="-150" xon="0" yon="0" width="768" height="200" swipeEnabled="y" background="top_chrome_bar.png">
		<togglebutton x="0" y="150" width="768" height="50" up="open_arrow.png" down="close_arrow.png" />
		<scrubber x="20" y="40" width="728" height="80" padding="4" centrePages="y" />
			
		<button x="20" y="0" width="100" height="20" up="close_issue.png" down="" pageSection="" tapEffectType="TAP_CLOSEISSUE" tapEffectData="" />
		<button x="200" y="0" width="100" height="20" up="home_up.png" down="home_down.png" pageSection="Home" tapEffectType="TAP_JUMPTOSECTION" tapEffectData="Home" />
		<button x="320" y="0" width="100" height="20" up="new_up.png" down="new_down.png" pageSection="New" tapEffectType="TAP_JUMPTOSECTION" tapEffectData="New" />
		<button x="440" y="0" width="100" height="20" up="next_up.png" down="next_down.png" pageSection="Next" tapEffectType="TAP_JUMPTOSECTION" tapEffectData="Next" />
		<button x="648" y="0" width="100" height="20" up="toc.png" down="" pageSection="" tapEffectType="TAP_TOC" tapEffectData="" />
	</bar>
</chrome>
