Importing Text

As seen in "HTML and CSS", story layers can be populated directly by simply typing in the text content in the story editor. This may well be sufficient for short, or static pieces of text. However, when dealing with content that has been commissioned from elsewhere, or content that will change from issue to issue, there are a couple of other ways by which text can be flowed into story layers.

Import body copy

Add source file

The first way is to flow in text from an external file inside the issue directory. Click on the Import body copy... button to bring up a list of all suitable text files within the issue and select the one you need, as illustrated in "Import body copy". This is essentially the same as opening the external file, and copying its content into the story editor and pressing save; subsequent changes to the content of the external file will not be reflected in the contents of the story layer.

The second way is to link the story layer to an external source file. The effect of this is that if the external file's content changes, this will be reflected in the story layer. This is a very powerful feature for pages where the layout remains fixed, but the content changes between issue to issue. See "Add source file"

Variables as source files

If you want to write the contents of a variable into your story box, specify the name of the variable in the Source File box, prefixed with a dollar sign. For example, to have the Score variable printed out, you would use $Score.

If you specify multiple variable names, separated by a comma, FutureFolio will automatically print both the name of the variable and its value, which makes it very easy to create shopping lists.

If you intend to use multiple variable names, you should be prepared to name your variables in a way that will be shown to users (eg "Meat and Dairy" rather than "meat_and_dairy"), and you should use the CSS styles p.FolioVariableName and p.FolioVariableValue to style the output.
