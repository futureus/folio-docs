Table Creation

To create a new table layer, click on the Add table button.

Adding a table

Data for each individual table is stored in its own file within your issue. These are created and managed by FutureFolio Creator via the table editor window.

Once you have created a new table layer, clicking on the layer settings button will bring up a dialog asking you how you want to configure your table - a new blank table ("Create blank table") or copied from another table in that issue ("Create from file").

If you choose to create a table from an existing one, you will be presented with a list of any table files found in that issue. Select the table you want to copy. FutureFolio Creator will copy the selected table to a new file and take you into the table editor where you can make any changes.

If you choose to create a blank template, Creator will take you directly into the table editor where you can set up your table from scratch.

Tables from older issues

If you create your issues by copying directly from an old issue, you may find that your list of existing tables is empty on first run. This is because the file format changed from .xml to a custom file type - .table - and the list is populated by .table files.

To migrate your old-style tables to the new format, simply open the table editor for each and click Save. You will then see them in the list.

Note that this migration to the new format is not compulsory and your old .xml files will still be displayed correctly.

If you choose to delete a table layer, Creator will ask you if you wish to delete the table's associated file. If you wish to use the table again later in the issue, choose "No". Otherwise, click "Yes" and the file will be deleted from your issue.

Note that table data files which you choose not to delete (and which are therefore no longer associated with a table layer), although remaining in your issue directory, will not be copied into your final built issue. This means that there are no negative side effects to not deleting them, other than cluttering your editing issue directory!

The data file associated with your table layer is shown across the top of the table editor window. You can change the data file which your table layer is associated with by clicking the File button. You will be presented with a list of the table files in that issue - select the file you want your table layer to use. If you select a different file to the one your table layer is currently using, Creator will ask you if you want to delete the file which the table used to reference. As before, select "No" if you want to use it again, and "Yes" if it is no longer needed.

Shared table files

Be careful when changing your table layer's associated data file. If you select a data file which is also referenced by another table layer in your issue, both table layers will then be referencing the same data file. This means that any changes you make to the table data will affect both table layers. It also means that if you choose to delete the table data file if and when one layer no longer needs it, the other layer which references that data file will be left without data and therefore be blank.
